# Plan 03-03: 数组化离散状态数据组织

<objective>
重写 `_organize_discrete_data()` 的内部实现，把当前 DataFrame 重构、pandas 索引和重复 token 解析改为更直接的 ndarray 流程，同时保证候选评分输入矩阵和状态条件学习结果完全一致。
</objective>

<context>
算法分析显示 `_organize_discrete_data()` 是当前最大的单文件热点。用户同意简化和优化该函数，但要求结果一致，并特别强调候选统计不能脱离当前 pair posterior 语义。因此本计划只优化状态/父子矩阵组织方式，不改变 BD score 公式、候选顺序或每轮重算状态依赖链接的语义。
</context>

<design_contract>

## 数据类设计

```python
@dataclass(frozen=True)
class DiscreteLearningData:
    """保存候选评分所需的离散父子变量、状态条件变量和状态依赖信息。"""
    data_parent: np.ndarray
    data_child: np.ndarray
    data_condition: np.ndarray
    condition_state: np.ndarray
    learned_state_adjacency: np.ndarray
```

## 函数接口设计

```python
def _organize_discrete_data(
    self,
    parsed: ParsedSequence,
    parent_token: str,
    child_token: str,
    state_values: np.ndarray,
    state_conditions: Sequence[Sequence[int]],
) -> DiscreteLearningData
```

- `state_values` 在核心内部使用 ndarray，列顺序由 `state_names` 明确约束。
- `data_parent` 与 `data_child` 必须继续使用当前 1/2 离散编码。
- `condition_state` 的行列和取值必须与当前状态条件学习过程一致。
</design_contract>

<threat_model>

| Threat | Severity | Mitigation |
|--------|----------|------------|
| ndarray 列顺序和 DataFrame 列名对应错误 | HIGH | 测试同时断言 `state_names` 顺序、矩阵形状和代表性行值。 |
| parent/child 离散编码 1/2 与当前实现相反 | HIGH | 过程测试固定 `data_parent`、`data_child` 的完整小样本数组。 |
| 状态条件链接学习被缓存或降频 | HIGH | 每轮仍调用 `learn_state_condition_links()`；测试断言调用结果和当前矩阵一致。 |
| 误把 pair posterior 改成 raw count | HIGH | 保持 scoring 调用顺序，过程测试继续断言 posterior 值。 |
</threat_model>
