# Plan 03-05: 重构 skip-gram 和输出适配边界

<objective>
在主学习流程稳定后，重构 `detect_skip_gram()` 的内部表达和正式输出边界，保持 `N` 映射、插入位置、候选评分和最终 skip-gram 结果完全一致，并确保旧格式转换继续只存在于验证适配器中。
</objective>

<context>
用户允许按执行方判断重构 `detect_skip_gram()`，但明确提醒 `N` 的映射方式、映射逻辑和插入回去的位置很容易改变结果。因此本计划把 skip-gram 作为主循环之后的单独风险面处理，同时检查正式输出是否仍被旧格式兼容字段污染。
</context>

<design_contract>

## skip-gram 过程接口

```python
@dataclass(frozen=True)
class SkipGramCandidateTrace:
    """保存一次 skip-gram 候选插入和评分所需的过程指标。"""
    parent_token: str
    child_token: str
    sequence_with_n: list[str]
    n_insert_positions: list[int]
    score_without_parent: float
    score_with_parent: float
    posterior: float
```

```python
def _build_skip_gram_sequence(
    self,
    parsed: ParsedSequence,
    parent_token: str,
    child_token: str,
) -> tuple[list[str], list[int]]
```

- 该 trace 只用于测试和解释关键过程，不进入正式输出。
- `sequence_with_n` 必须保留当前 `N` 插入位置语义。
- 正式输出仍由 `build_structured_output()` 组装为新结构；旧字段由 `legacy_adapter.py` 重建。
</design_contract>

<threat_model>

| Threat | Severity | Mitigation |
|--------|----------|------------|
| `N` 插入位置偏移一位导致最终结果仍部分相似但过程错误 | HIGH | 过程测试逐候选断言 `sequence_with_n` 和 `n_insert_positions`。 |
| skip-gram posterior 误用纯频次或错误条件矩阵 | HIGH | 断言候选评分矩阵和 posterior 关键值。 |
| 为旧格式 `gram` 保留核心冗余字段 | MEDIUM | 旧格式字段重建只允许在 `legacy_adapter.py`。 |
| 输出字段重命名破坏验证适配器 | HIGH | 同步更新验证适配器测试和全量验证。 |
</threat_model>
